myMovie

This is a build in progress of a movie API. Users will eventually be able to use this app to receive any information that a user might want about movies.

Endpoint 1

This endpoint will return a full list of movie titles and the URL is /movies. This is a get request will return a JSON object

Endpoint 2

Will return all data on a single movie. The URL is /movies/[movie title] with [movie title] being the title searched for. This is a get request and the response will be a JSON object

Endpoint 3

Will return a description of specific genres. The URL is /movies/genre/[name] with [name] being a specific name of a genre. this is a get request and the response will be in text format

Endpoint 4

Will return data about a director. The url is /directors/[name] with [name] being a specific director. This is a get request and the response will be in text format.

Endpoint 5

This will allow users to register on the the app. The URL is /users. The request body is a JSON object required to be in this format:
{userName: "User_name",
password: "*********",
email:"user@mail.com",
birthday: "mm/dd/yyyy"}
This is a post request and the response will be in text format confirming that they have registered

Endpoint 6

This will allow users to update their email and birthday. The URL is /users/[userName] with [userName] being the users user name. The request body is json object in this format:
{userName: "User_name",
password: "*********",
email:"user@mail.com",
birthday: "mm/dd/yyyy"}
This is a PUT request and the response will be in text format confirming that they have succesfully updated.

Endpoint 7

This will allow users to add a movie to their favorites list. The URL is /users/[userName]/favorites/[movie] with [userName] being the user name and [movie] being the movie's ID to be added to the favorite list. This is a PUT request and the response will be in JSON format containing a list of all of the Users favorite movies

Endpoint 8

This will allow users to remove a movie from their favorites list. The URL is /users/[userName]/favorites/[movie] with [userName] being the user name and [movie] being the movie's ID to remove. This is DELETE request and the response will be in JSON format containing the updated favorites list.

Endpoint 9

This will allow the users to deregister from the app. The URL is /users/[userName] with [userName] users user name. This is a DELETE request and the response will be a text message confirming that they have succesfully unregistered from the app.
